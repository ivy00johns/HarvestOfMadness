The goal, in one paragraph

Make Harvest of Madness read like Stanford's Smallville: a dense, organic town where a believable population lives — visiting real places, talking, gossiping, planning gatherings, farming, feuding — and where the player can read what's happening without drowning in text. The bar is density and legibility, not map size or new assets. We already own every asset we need. The recurring failure has been placement and presentation, not capability.

Why we kept stalling (so we don't repeat it)

Three rounds of the same complaint, because we kept fixing the wrong layer.

RoundThe complaintReal causeLesson
1Houses identical, veg used as fences, agents conga-line, "can't read this madness" In-world text soup + a decor hash planting produce in columns Words don't belong in the world. Smallville shows only emoji/initials; sentences live in panels.
2"I liked the original cards, I just couldn't access them — it can scroll" We replaced rich cards with compact chips Don't trade richness for density — make richness scrollable.
3"Still 25 agents, same houses, no stores with aisles. Did you look at their code?" We studied the renderer, not the map data the_ville is 140×100, ~1,360 furniture tiles in large multi-room buildings (~9× density). Bigger, packed buildings — not more grass.

The load-bearing insight: furniture is render-only — passability is tile-type-driven, so furniture sprites never block pathfinding. Interiors can be packed arbitrarily dense with zero pathfinding or test risk. This is the lever for "alive".

Hard constraints (never violate)

  • Determinism — no Math.random, no Date in map/sim generation. Re-running generateMap() is byte-identical.
  • Tests stay green (~1073). Map structure/counts/connectivity, personas, party-emergence, observation, integration. TDD-converge coordinates — the tests are the source of truth.
  • Reachability — every door → tavern paths through passable tiles within the budget (40 → 100). Plots ≤ Chebyshev-4 of doors.
  • Assets — owned/credited only (CC0/CC-BY-SA, attributed in CREDITS.md and CREDITS.txt; the manifest test reads .txt). No Smallville commercial tilesets.
  • Furniture is render-only — exploit it for density; it can't break paths.
  • Toolchain — nvm shadows binaries; use absolute node paths; tests via node_modules/vitest/vitest.mjs run, tsc via node_modules/typescript/bin/tsc --noEmit.
  • Commits — only when verified; --no-gpg-sign.
